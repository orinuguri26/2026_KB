1. CSS 가 왜 필요한가? 
   Cascading Style Sheets 약자 
   HTML이 정보를 표현하는 역할 - 구조 
   CSS는 HTML 문서를 시각적으로 꾸미는 역할 - (예쁘게 꾸미기)
   JavaScript 정적인 HTML 문서에 동적인 행위를 부여하는 역할 (동작)
   --- 가독성과 코드관리 효율성을 높이고
       재사용성과 독립성도 보장 

2. CSS  적용 
  - HTML => 
  <style>
            p(선택자) {
             color: blue;
          }
           </style>

  - 외부 css파일 작성 => html문서 <link>태그 이용 적용
3. 외부 CSS 파일 적용 
4. 글씨 크기, 정렬 
   google 검색시 => css text-size(align) property 키워드 => MDN 
   - 폰트 크기 지정 속성 : font-size 
   - 정렬 속성 : text-align : left, center, right, justify (속성값)

5. 선택자 (Selector)
   HTML에 스타일을 적용할때, HTML요소를 지정하는 역할
   
   내가 작성한 모든 태그에 스타일을 적용하려면
   일일이 태그를 적어야만 할까? 

   내가 작성한 <p> 첫 번째에만 원하는 CSS를 할 수 없을까?

   - 전체 선택자 : * 
       * { 
        ....
       }
   - 유형(태그) 선택자 : h1, p, img  특정태그를 지정하는 방식
     
   - 아이디 선택자 
     아이디로 요소(태그)를 식별해 주는 선택자 
     특정 아이디로 하나를 선택할때 사용 

   - 클래스 선택자
     특정 클래스에 속하는 요소를 선택한다. 
     하나 이상의 스타일을 지정할때 사용한다.
   - 복합선택자 
     부모와 자식태그가 복합적으로 쓰인다 라고 하여
     복합선택자라고 명칭한다. 

     서로의 관계와 위치를 유용하게 결합하는 방식을 제공

     서로 => 자손 선택자 
             자식 선택자
             인접 형제 선택자 
             일반 형제 선택자

CSS 배치 속성  ,  반응형 디자인 

1. 배치 속성이 뭐임? 
   display ==> block, inline    
2. 박스모델 (box model) : 마진,테두리,패딩 
   정의 : 기본적으로 모든 HTML 요소를 감싸는 상자  
   구성 : 요소(element), 패딩(padding), 테두리(border),마진(margin)

   블록과 인라인이라는 개념은 왜 존재할까? 
   보기 좋게 배치하기 위해서 
   요소를 원하는 위치에 배치하려면 태그마다 특징을 알아야한다.

   - block
   - inline
   - box model
